# HVV Traffic Analyzer — 使用手册

> 本文档用实际例子告诉你：不加选项会怎样，加了选项会怎样，出错了会提示什么。

---

## 一、最快上手

### 第一步：确认环境

```bash
cd D:\Gamecreate\hw
pip install -r requirements.txt
tshark --version          # 确认 tshark 可用
```

### 第二步：启动（交互式菜单，推荐）

```bash
python main.py
```

启动后会先显示**免责声明**，按任意键确认后进入**交互式菜单**，全程用数字选项和上下键操作，无需记忆任何命令行参数。

### 第三步：或使用 CLI 模式直接运行（管理员权限）

```bash
python main.py -i "以太网"
```

### 2.4 子菜单一览

#### [1] 网卡选择
上下键选择网卡，Enter 确认。

#### [2] 过滤设置
```
  ┌────────────────────────────────────────────────────────────────────────────────────────────────────┐
  │                                                                                                    │
  │  [1] BPF 过滤表达式                                                                              ❌ 无 │

  └────────────────────────────────────────────────────────────────────────────────────────────────────┘
```
输入数字后按提示输入值，`[C]` 一键清除所有过滤。

#### [3] 检测设置
- `[1]` 切换攻击检测开/关
- `[2]` 选择检测类别（上下键移动，空格切换，A全选，N全不选，Enter确认）
- `[3]` 指定自定义规则文件路径

可选类别：`sql_injection` `xss` `cmd_injection` `dir_traversal` `file_upload` `deserialization` `webshell` `c2` `log4shell`

#### [4] IOC 设置
- `[1]` 切换 IOC 提取开/关
- `[2]` 指定 IOC 白名单文件
- `[3]` 切换 IOC 导出开/关

#### [5] 可疑流量设置
- `[1]` 切换可疑流量检测开/关
- `[2]` 切换可疑流量导出开/关
- `[3]` 指定自定义可疑规则文件

#### [6] 输出设置
- `[1]` 切换日志文件开/关
- `[2]` 切换静默模式
- `[3]` 切换彩色输出
- `[4]` 设置统计摘要间隔（秒）
- `[5]` 设置抓包时长（秒，留空为持续运行）

## 四、默认行为详解

使用菜单模式或运行 `python main.py -i "以太网"` 时（不加任何其他选项）：

### ✅ 默认开启

| 功能 | 说明 |
|------|------|
| 全量抓包 | 抓取网卡上所有流量，不做任何过滤 |
| 确认攻击检测 | 18 条规则，9 大类全部启用 |
| 可疑流量检测 | 16 条规则全部启用，终端黄色显示 |
| 定期统计摘要 | 每 5 分钟自动输出一次 |
| Ctrl+C 最终报告 | 停止时输出完整统计 |

### ❌ 默认关闭（需通过菜单或加选项启用）

| 功能 | 说明 | 菜单 | CLI |
|------|------|------|-----|
| 可疑流量导出文件 | 只在终端显示，不写文件 | [5] → [2] | `--suspicious-export` |
| IOC 自动提取 | 不提取威胁指标 | [4] → [1] | `--extract-ioc` |
| IOC 导出文件 | 不写 IOC 文件 | [4] → [3] | `--ioc-export` |
| 日志文件 | 终端输出不保存 | [6] → [1] | `-o` |
| 流量过滤 | 抓所有包 | [2] → 各子项 | `-f` / `--src-ip` 等 |
| 限定检测类别 | 9 类全检 | [3] → [2] | `--detect webshell,c2` |
| 限定时长 | 持续运行 | [6] → [5] | `--duration 3600` |

## 五、参数错误提示（CLI 模式）

所有参数错误都会给出清晰提示，不会出现 Python 报错：

### 无效 IP

```bash
python main.py -i "以太网" --src-ip "abc"
```
```
错误: 源 IP (--src-ip) 格式无效: 'abc'
  正确格式: 10.0.0.1 或 192.168.0.0/16
```

### 无效检测类别

```bash
python main.py -i "以太网" --detect "wrong_category"
```
```
错误: 无效的检测类别: wrong_category
  可选类别: c2, cmd_injection, deserialization, dir_traversal, file_upload, log4shell, sql_injection, webshell, xss
```

### 无效端口

```bash
python main.py -i "以太网" --port 99999
```
```
错误: 端口号无效: 99999（有效范围: 1-65535）
```

### 无效正则表达式

```bash
python main.py -i "以太网" --content "[invalid"
```
```
错误: --content 正则表达式语法无效: unterminated character set at position 0
```

### 不存在的规则文件

```bash
python main.py -i "以太网" --rules "no_such_file.yaml"
```
```
错误: 确认攻击规则 (--rules) 文件不存在: no_such_file.yaml
```

## 六、自动路径功能（CLI 模式）

以下三个选项支持**不指定路径**，自动生成带时间戳的文件名：

| 选项 | 不加 | 加但不写路径 | 加且写路径 |
|------|------|-------------|-----------|
| `-o` | 不保存日志 | 自动生成 `capture_20260619_143000.log` | 保存到你指定的路径 |
| `--suspicious-export` | 不导出可疑流量 | 自动生成 `suspicious_traffic_20260619_143000.txt` | 导出到你指定的路径 |
| `--ioc-export` | 不导出 IOC | 自动生成 `ioc_20260619_143000.json` | 导出到你指定的路径 |

### 示例

```bash
# 自动生成 3 个文件
python main.py -i "以太网" -o --suspicious-export --ioc-export --extract-ioc

# 生成的文件：
#   capture_20260619_143000.log              ← 终端日志
#   capture_20260619_143000_report.json      ← 详细报告（随 -o 自动生成）
#   suspicious_traffic_20260619_143000.txt   ← 可疑流量
#   ioc_20260619_143000.json                 ← IOC 数据
```

## 七、各选项使用示例（CLI 模式）

### 7.1 `--suspicious-export` — 导出可疑流量

```bash
python main.py -i "以太网" --suspicious-export
python main.py -i "以太网" --suspicious-export my_suspicious.txt
```

导出文件每行一条，格式：
```
时间 | 严重度 | src:port -> dst:port | 协议 | 指标摘要 | Wireshark过滤器 | grep关键字
```

**用法**：复制 Wireshark 过滤器列，粘贴到 Wireshark 过滤栏，直接定位数据包。

---

### 7.2 `--extract-ioc` + `--ioc-export` — IOC 提取和导出

```bash
python main.py -i "以太网" --extract-ioc
python main.py -i "以太网" --extract-ioc --ioc-export
python main.py -i "以太网" --extract-ioc --ioc-export ioc_report.json
```

---

### 7.3 `-f` — BPF 过滤（底层过滤，最高效）

```bash
# 只抓 HTTP + HTTPS + DNS
python main.py -i "以太网" -f "port 80 or port 443 or port 53"

# 只抓到某服务器的流量
python main.py -i "以太网" -f "host 192.168.1.100"

# 横向移动常用端口
python main.py -i "以太网" -f "port 445 or port 3389 or port 5985"
```

---

### 7.4 `--src-ip` / `--dst-ip` — IP 过滤

```bash
python main.py -i "以太网" --src-ip 10.0.0.50
python main.py -i "以太网" --dst-ip 192.168.1.0/24
```

---

### 7.5 `--detect` — 限定攻击检测类别

```bash
# 只检测 Webshell + C2
python main.py -i "以太网" --detect webshell,c2

# 只检测 SQL 注入
python main.py -i "以太网" --detect sql_injection
```

---

### 7.6 `--no-detect` / `--no-suspicious` — 关闭检测

```bash
# 只看可疑流量，不看确认攻击
python main.py -i "以太网" --no-detect --suspicious-export

# 只看确认攻击，不看可疑流量
python main.py -i "以太网" --no-suspicious
```

---

### 7.7 `-o` — 保存日志

```bash
python main.py -i "以太网" -o
python main.py -i "以太网" -o capture_log.txt
```

---

### 7.8 `--duration` — 限定运行时长

```bash
python main.py -i "以太网" --duration 3600    # 1 小时
python main.py -i "以太网" --duration 600     # 10 分钟
```

---

### 7.9 `--quiet` — 静默模式

```bash
python main.py -i "以太网" --quiet
```

只输出告警和可疑流量，不输出 INFO 日志。

## 八、实战场景

### 场景 1：护网全面监控（推荐）

**菜单模式**：启动 → 菜单中依次配置 → `S` 开始  
**CLI 模式**：
```bash
python main.py -i "以太网" --suspicious-export --extract-ioc --ioc-export --duration 28800
```

### 场景 2：定向排查可疑 IP

**菜单模式**：[2] 过滤设置 → [2] 源 IP 输入 `10.0.0.50` → [5] 开启导出 → `S`  
**CLI 模式**：
```bash
python main.py -i "以太网" --src-ip 10.0.0.50 --suspicious-export
```

### 场景 3：只看 Webshell 和 C2

**菜单模式**：[3] 检测设置 → [2] 只选 webshell 和 c2 → `S`  
**CLI 模式**：
```bash
python main.py -i "以太网" --detect webshell,c2 --no-suspicious
```

### 场景 4：检测内网横向移动

```bash
python main.py -i "以太网" -f "port 445 or port 3389 or port 5985 or port 135" --suspicious-export
```

### 场景 5：监控 Web 服务器

```bash
python main.py -i "以太网" -f "port 80 or port 443 or port 8080" --suspicious-export --extract-ioc
```

### 场景 6：用导出的 Wireshark 过滤器回溯

```bash
# 复制导出文件中的 Wireshark 过滤器，粘贴到 Wireshark
# 或用 tshark 命令行：
tshark -r capture.pcap -Y "ip.addr==10.0.0.51 && tcp.port==8080 && http.request.uri contains /actuator"
```
